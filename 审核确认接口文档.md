# 审核确认接口文档

## 概述

本文档描述了制度审查智能体的审核确认相关接口，用于审核人员对审查结果进行确认、驳回等操作。

## 认证

所有接口均需要 JWT Token 认证，请在请求头中携带：

```
Authorization: Bearer <token>
```

## 状态说明

### 审查结果状态 (ResultStatus)

| 状态值 | 说明 |
|--------|------|
| `pending_review` | 待审核 |
| `reviewing` | 审核中 |
| `completed` | 已完成 |

### 问题状态 (IssueStatus)

| 状态值 | 说明 |
|--------|------|
| `pending` | 待处理 |
| `accepted` | 已接受 |
| `rejected` | 已拒绝 |
| `partial_accepted` | 部分接受 |

---

## 接口列表

### 1. 开始审核

将审查结果状态从 `pending_review` 改为 `reviewing`，并设置审核人ID。

**请求**

```
POST /api/audit/result/{result_id}/start-review
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

**响应示例**

```json
{
  "success": true,
  "message": "已开始审核",
  "result": {
    "id": 1,
    "task_id": 1,
    "document_name": "测试文档.docx",
    "risk_level": "medium",
    "total_issues": 5,
    "compliance_issues": 2,
    "consistency_issues": 2,
    "format_issues": 1,
    "status": "reviewing",
    "created_at": "2026-04-07T10:00:00",
    "updated_at": "2026-04-07T10:30:00"
  }
}
```

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 400 | 无法开始审核，结果不存在或状态不正确 |
| 401 | 未授权，请先登录 |

---

### 2. 确认审查结果

确认审查结果，将状态改为 `completed`，完成审核流程。

**请求**

```
POST /api/audit/result/{result_id}/confirm
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

**请求体**

```json
{
  "comment": "审核通过，无问题"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| comment | string | 否 | 审核意见/备注 |

**响应示例**

```json
{
  "success": true,
  "message": "审查结果已确认",
  "result": {
    "id": 1,
    "task_id": 1,
    "document_name": "测试文档.docx",
    "risk_level": "medium",
    "total_issues": 5,
    "compliance_issues": 2,
    "consistency_issues": 2,
    "format_issues": 1,
    "status": "completed",
    "created_at": "2026-04-07T10:00:00",
    "updated_at": "2026-04-07T11:00:00"
  }
}
```

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 404 | 结果不存在 |
| 401 | 未授权，请先登录 |

---

### 3. 驳回审查结果

驳回审查结果，状态重置为 `pending_review`，需要重新审查。

**请求**

```
POST /api/audit/result/{result_id}/reject
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

**请求体**

```json
{
  "reason": "审查结果不完整，需要重新审查"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| reason | string | 是 | 驳回原因 |

**响应示例**

```json
{
  "success": true,
  "message": "审查结果已驳回",
  "result": {
    "id": 1,
    "task_id": 1,
    "document_name": "测试文档.docx",
    "risk_level": "medium",
    "total_issues": 5,
    "status": "pending_review",
    "created_at": "2026-04-07T10:00:00",
    "updated_at": "2026-04-07T11:30:00"
  }
}
```

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 404 | 结果不存在 |
| 401 | 未授权，请先登录 |

### 4. 获取审核统计

获取审核进度统计信息，包括各状态问题数量和审核进度百分比。

**请求**

```
GET /api/audit/result/{result_id}/review-statistics
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

**响应示例**

```json
{
  "total_issues": 10,
  "pending_issues": 3,
  "accepted_issues": 5,
  "rejected_issues": 2,
  "partial_accepted_issues": 0,
  "review_progress": 70,
  "all_reviewed": false
}
```

**响应字段说明**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| total_issues | int | 问题总数 |
| pending_issues | int | 待处理问题数 |
| accepted_issues | int | 已接受问题数 |
| rejected_issues | int | 已拒绝问题数 |
| partial_accepted_issues | int | 部分接受问题数 |
| review_progress | int | 审核进度百分比 (0-100) |
| all_reviewed | bool | 是否全部问题已审核 |

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 404 | 结果不存在 |
| 401 | 未授权，请先登录 |

---

### 5. 批量更新问题状态

批量更新多个问题的状态。

**请求**

```
POST /api/audit/result/{result_id}/issues/batch
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

**请求体**

```json
{
  "issue_ids": [1, 2, 3],
  "status": "accepted"
}
```

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| issue_ids | int[] | 是 | 问题ID列表 |
| status | string | 是 | 目标状态：`accepted`/`rejected`/`partial_accepted` |

**响应示例**

```json
{
  "success": true,
  "updated_count": 3,
  "message": "成功更新 3 个问题状态"
}
```

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 400 | 无效的状态值 或 批量更新失败 |
| 401 | 未授权，请先登录 |

---

### 6. 接受所有问题

一键接受该结果下的所有问题。

**请求**

```
POST /api/audit/result/{result_id}/issues/accept-all
```

**路径参数**

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| result_id | int | 是 | 审查结果ID |

```json
{

  "updated_count": 10,

  "message": "没有需要处理的问题"
}
```

**错误响应**

| 状态码 | 说明 |
|--------|------|
| 401 | 未授权，请先登录 |

## 状态流转图

### 审查结果状态流转

```
                    ┌─────────────────┐
                    │                 │
                    ▼                 │
pending_review ──────► reviewing ──────► completed
     │                                    ▲
     │                                    │
     └────────────────────────────────────┘
                  (驳回后重新审核)
```

### 问题状态流转

```
                    ┌──────────────────┐
                    │                  │
                    ▼                  │
              ┌──────────┐             │
              │ pending  │─────────────┤
              └──────────┘             │
                    │                  │
        ┌───────────┼───────────┐      │
        ▼           ▼           ▼      │
  ┌──────────┐ ┌──────────┐ ┌──────────┴───┐
  │ accepted │ │ rejected │ │ partial_     │
  │          │ │          │ │ accepted     │
  └──────────┘ └──────────┘ └──────────────┘
```

## 审核轨迹记录

所有审核操作都会自动记录到 `audit_trails` 表中，包括：

| 操作 | 记录内容 |
|------|----------|
| 开始审核 | 审核人开始审核审查结果 |
| 确认审查结果 | 审查结果已确认，审核意见：xxx |
| 驳回审查结果 | 驳回原因：xxx |
| 批量更新问题状态 | 批量将 N 个问题状态更新为 xxx |

## 使用示例

### 完整审核流程

1. **开始审核**
```bash
curl -X POST "http://localhost:8000/api/audit/result/1/start-review" \
  -H "Authorization: Bearer <token>"
```

2. **查看审核统计**
```bash
curl -X GET "http://localhost:8000/api/audit/result/1/review-statistics" \
  -H "Authorization: Bearer <token>"
```

3. **逐个审核问题**
```bash
# 接受问题
curl -X POST "http://localhost:8000/api/audit/issue/1/accept?suggestion=建议修改为..." \
  -H "Authorization: Bearer <token>"

# 拒绝问题
curl -X POST "http://localhost:8000/api/audit/issue/2/reject?reject_reason=误报" \
  -H "Authorization: Bearer <token>"
```

4. **或批量处理问题**
```bash
curl -X POST "http://localhost:8000/api/audit/result/1/issues/batch" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"issue_ids": [1, 2, 3], "status": "accepted"}'
```

5. **确认审查结果**
```bash
curl -X POST "http://localhost:8000/api/audit/result/1/confirm" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"comment": "审核完成，所有问题已处理"}'
```

### 快速审核流程

如果所有问题都认可，可以简化流程：

```bash
# 1. 接受所有问题
curl -X POST "http://localhost:8000/api/audit/result/1/issues/accept-all" \
  -H "Authorization: Bearer <token>"

# 2. 确认审查结果
curl -X POST "http://localhost:8000/api/audit/result/1/confirm" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '{"comment": "审核通过"}'
```
