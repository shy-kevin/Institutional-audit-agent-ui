## 认证说明

### 认证方式

系统使用 JWT (JSON Web Token) 进行身份认证。

### 请求头

所有需要认证的接口都需要在请求头中携带 Token：

```
Authorization: Bearer <access_token>
```

### 获取 Token

通过登录接口获取 Token，Token 有效期为 24 小时。

## 用户注册

### POST `/api/auth/register`

创建新用户账号。

**请求方式**: `POST`

**Content-Type**: `application/json`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| username | string | 是 | 用户名（2-50字符） |
| account | string | 是 | 登录账号（3-50字符，唯一） |
| password | string | 是 | 密码（6-50字符） |
| phone | string | 否 | 手机号 |
| department | string | 否 | 部门 |

**请求示例**:

```json
{
    "username": "张三",
    "account": "zhangsan",
    "password": "123456",
    "phone": "13800138000",
    "department": "技术部"
}
```

**响应示例**:

```json
{
    "id": 2,
    "username": "张三",
    "account": "zhangsan",
    "phone": "13800138000",
    "department": "技术部",
    "role": "user",
    "is_active": true
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 400 | 账号已存在 |

## 用户登录

### POST `/api/auth/login`

验证账号密码，返回访问令牌。

**请求方式**: `POST`

**Content-Type**: `application/json`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| account | string | 是 | 登录账号 |
| password | string | 是 | 密码 |

**请求示例**:

```json
{
    "account": "admin",
    "password": "admin123"
}
```

**响应示例**:

```json
{
    "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
    "token_type": "bearer",
    "user": {
        "id": 1,
        "username": "admin",
        "account": "admin",
        "phone": null,
        "department": "System",
        "role": "admin",
        "is_active": true
    }
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 401 | 账号或密码错误 |

## 获取当前用户信息

### GET `/api/auth/me`

获取当前登录用户的详细信息。

**请求方式**: `GET`

**请求头**:

```
Authorization: Bearer <access_token>
```

**响应示例**:

```json
{
    "id": 1,
    "username": "admin",
    "account": "admin",
    "phone": null,
    "department": "System",
    "role": "admin",
    "is_active": true,
    "last_login": "2026-04-07T10:30:00",
    "created_at": "2026-04-07T09:00:00"
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 401 | 未登录或令牌无效 |

**请求头**:

| 状态码 | 说明 |

## 获取用户列表（管理员）

### GET `/api/auth/list`

获取系统中所有用户的列表，需要管理员权限。

**请求方式**: `GET`

**请求头**:

```
Authorization: Bearer <access_token>
```

**查询参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| skip | int | 否 | 跳过的记录数，默认0 |
| limit | int | 否 | 返回的最大记录数，默认100 |
| role | string | 否 | 按角色筛选（admin/user） |
| department | string | 否 | 按部门筛选 |

**请求示例**:

```
GET /api/auth/list?skip=0&limit=10&role=user
```

**响应示例**:

```json
{
    "total": 5,
    "items": [
        {
            "id": 2,
            "username": "张三",
            "account": "zhangsan",
            "phone": "13800138000",
            "department": "技术部",
            "role": "user",
            "is_active": true,
            "last_login": null,
            "created_at": "2026-04-07T10:00:00"
        }
    ]
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 401 | 未登录 |
| 403 | 需要管理员权限 |

## 重置用户密码（管理员）

### PUT `/api/auth/{user_id}/reset-password`

重置指定用户的密码，需要管理员权限。

**请求方式**: `PUT`

**请求头**:

```
Authorization: Bearer <access_token>
```

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_id | int | 是 | 用户ID |

**查询参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| new_password | string | 是 | 新密码 |

**请求示例**:

```
PUT /api/auth/2/reset-password?new_password=newpass123
```

**响应示例**:

```json
{
    "message": "密码重置成功"
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 401 | 未登录 |
| 403 | 需要管理员权限 |
| 404 | 用户不存在 |

## 启用/禁用用户（管理员）

### PUT `/api/auth/{user_id}/status`

启用或禁用指定用户，需要管理员权限。

**请求方式**: `PUT`

**请求头**:

```
Authorization: Bearer <access_token>
```

**路径参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| user_id | int | 是 | 用户ID |

**查询参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| is_active | bool | 是 | 是否激活（true启用，false禁用） |

**请求示例**:

```
PUT /api/auth/2/status?is_active=false
```

**响应示例**:

```json
{
    "message": "用户状态更新成功"
}
```

**错误响应**:

| 状态码 | 说明 |
|--------|------|
| 401 | 未登录 |
| 403 | 需要管理员权限 |
| 404 | 用户不存在 |

## 用户角色说明

| 角色 | 值 | 说明 |
|------|-----|------|
| 管理员 | admin | 拥有所有权限，可以管理用户 |
| 普通用户 | user | 只能管理自己的信息，使用审查功能 |

## 错误响应

### 401 未认证

```json
{
    "detail": "未登录"
}
```

或

```json
{
    "detail": "令牌无效或已过期"
}
```

### 403 权限不足

```json
{
    "detail": "需要管理员权限"
}
```

### 404 资源不存在

```json
{
    "detail": "用户不存在"
}
```

## 审查接口认证要求

所有审查相关接口现在都需要用户认证：

| 接口 | 说明 |
|------|------|
| GET /api/audit/statistics | 获取审查统计 |
| GET /api/audit/tasks | 获取任务列表 |
| GET /api/audit/history | 获取审查历史 |
| POST /api/audit/task/create | 创建审查任务 |
| GET /api/audit/task/{task_id} | 获取任务详情 |
| POST /api/audit/task/{task_id}/start | 启动审查 |
| POST /api/audit/task/{task_id}/pause | 暂停审查 |
| POST /api/audit/task/{task_id}/cancel | 取消审查 |
| GET /api/audit/task/{task_id}/stream | 流式审查结果 |
| GET /api/audit/result/{result_id} | 获取审查结果 |
| POST /api/audit/result/{result_id}/export | 导出报告 |

审查任务创建时会自动关联当前登录用户（`user_id`），统计数据和历史记录也只显示当前用户的数据。

## 前端集成示例

### 登录并保存 Token

```javascript
// 登录
const loginResponse = await fetch('/api/auth/login', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ account: 'admin', password: 'admin123' })
});

const loginData = await loginResponse.json();

// 保存 Token
localStorage.setItem('access_token', loginData.access_token);
localStorage.setItem('user_info', JSON.stringify(loginData.user));
```

### 携带 Token 请求

```javascript
const token = localStorage.getItem('access_token');

const response = await fetch('/api/audit/statistics', {
    headers: {
        'Authorization': `Bearer ${token}`
    }
});

if (response.status === 401) {
    // Token 过期，跳转登录
    window.location.href = '/login';
}
```

### Axios 拦截器配置

```javascript
import axios from 'axios';

const api = axios.create({
    baseURL: '/api'
});

// 请求拦截器 - 添加 Token
api.interceptors.request.use(config => {
    const token = localStorage.getItem('access_token');
    if (token) {
        config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
});

// 响应拦截器 - 处理认证错误
api.interceptors.response.use(
    response => response,
    error => {
        if (error.response?.status === 401) {
            localStorage.removeItem('access_token');
            window.location.href = '/login';
        }
        return Promise.reject(error);
    }
);

export default api;
```
