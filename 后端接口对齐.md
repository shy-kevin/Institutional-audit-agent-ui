# 制度审查智能体 - 后端接口对齐文档

## 基础信息

- **API基础地址**: `http://localhost:8000`
- **数据格式**: JSON
- **流式响应**: SSE (Server-Sent Events)

---

## 一、工作台页面 (Dashboard)

### 1.1 获取审查统计数据

**接口**: `GET /api/audit/statistics`

**描述**: 获取工作台展示的统计数据

**请求参数**: 无

**返回数据**:
```json
{
  "today_count": 10,           // 今日审查数
  "batch_count": 5,            // 批量审查数
  "risk_count": 3,             // 风险问题数
  "completed_count": 100,      // 已完成审查数
  "compliance_ratio": 85,      // 合规性审查占比 (%)
  "consistency_ratio": 10,     // 一致性审查占比 (%)
  "format_ratio": 5            // 形式审查占比 (%)
}
```

### 1.2 获取审查任务列表

**接口**: `GET /api/audit/tasks`

**描述**: 获取审查任务列表（用于展示待处理任务）

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| limit | number | 否 | 返回数量限制 |
| status | string | 否 | 任务状态筛选 |

**返回数据**:
```json
{
  "total": 50,
  "items": [
    {
      "id": 1,
      "document_name": "员工考勤管理制度.docx",
      "document_path": "/uploads/2024/01/xxx.docx",
      "status": "pending",           // pending | parsing | analyzing | completed | failed
      "audit_type": "draft",         // draft | revision | current
      "progress": 0,                 // 进度百分比 0-100
      "created_at": "2024-01-15T10:30:00Z",
      "updated_at": "2024-01-15T10:30:00Z",
      "completed_at": null
    }
  ]
}
```

### 1.3 获取审查历史记录

**接口**: `GET /api/audit/history`

**描述**: 获取审查历史记录列表

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| date_range | string | 否 | 日期范围筛选 |
| audit_type | string | 否 | 审查类型筛选 |
| risk_level | string | 否 | 风险等级筛选 |
| keyword | string | 否 | 关键词搜索 |

**返回数据**:
```json
{
  "total": 100,
  "items": [
    {
      "id": 1,
      "document_name": "财务报销制度.pdf",
      "audit_type": "draft",         // draft | revision | current
      "audit_time": "2024-01-15T10:30:00Z",
      "risk_level": "high",          // high | medium | low
      "issue_count": 15,
      "status": "completed",         // completed | archived
      "auditor": "张三"
    }
  ]
}
```

## 二、文档上传页面 (DocumentUploadPage)

### 2.1 上传文件

**接口**: `POST /api/knowledge-base/upload-file`

**描述**: 上传制度文档文件

**请求方式**: `multipart/form-data`

**请求参数**:
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| file | File | 是 | 文件对象 |

**返回数据**:
```json
{
  "file_path": "/uploads/2024/01/xxx.docx",
  "file_name": "员工考勤管理制度.docx",
  "file_size": 102400
}
```

### 2.2 创建审查任务

**接口**: `POST /api/audit/task/create`

**描述**: 创建新的审查任务

**请求参数**:
```json
{
  "document_path": "/uploads/2024/01/xxx.docx",
  "document_name": "员工考勤管理制度.docx",
  "audit_type": "draft",           // draft | revision | current
  "config_id": 1                   // 可选，审查配置ID
}
```

**返回数据**:
```json
{
  "id": 1,
  "document_name": "员工考勤管理制度.docx",
  "document_path": "/uploads/2024/01/xxx.docx",
  "status": "pending",
  "audit_type": "draft",
  "progress": 0,
  "created_at": "2024-01-15T10:30:00Z",
  "updated_at": "2024-01-15T10:30:00Z"
}
```

**接口**: `POST /api/audit/version-compare/create`

**描述**: 创建新旧版本制度比对任务

**请求参数**:
```json
{
  "old_document_path": "/uploads/2024/01/old.docx",
  "new_document_path": "/uploads/2024/01/new.docx",
```

## 四、审查进度页面 (AuditProgressPage)

### 4.1 获取审查任务详情

**接口**: `GET /api/audit/task/{id}`

**描述**: 获取审查任务详情（同3.2）

### 4.2 启动审查

**接口**: `POST /api/audit/task/{id}/start`

**描述**: 启动审查任务

**请求参数**:
```json
{
  "config_id": 1
}
```

**返回数据**:
```json
{
  "success": true,
  "message": "审查任务已启动"
}
```

### 4.3 暂停审查任务

**接口**: `POST /api/audit/task/{id}/pause`

**描述**: 暂停正在进行的审查任务

**请求参数**: 无

**返回数据**:
```json
{
  "success": true,
  "message": "审查任务已暂停"
}
```

### 4.4 取消审查任务

**接口**: `POST /api/audit/task/{id}/cancel`

**描述**: 取消审查任务

**请求参数**: 无

**返回数据**:
```json
{
  "success": true,
  "message": "审查任务已取消"
}
```

### 4.5 流式获取审查结果

**接口**: `GET /api/audit/task/{id}/stream`

**描述**: 流式获取审查过程中的实时结果

**请求参数**: 路径参数 `id` - 任务ID

**响应格式**: SSE (Server-Sent Events)

**返回数据** (每条SSE消息):
```
data: {"content": "正在解析文档...", "is_end": false}
data: {"content": "发现合规性问题：...", "is_end": false}
data: {"content": "审查完成", "is_end": true}
```

## 错误响应格式

所有接口在发生错误时应返回统一格式：

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "参数验证失败",
    "details": {
      "field": "document_path",
      "reason": "文件路径不能为空"
    }
  }
}
```

### 常见错误码

| 错误码 | HTTP状态码 | 说明 |
|--------|-----------|------|
| VALIDATION_ERROR | 400 | 参数验证失败 |
| UNAUTHORIZED | 401 | 未授权 |
| FORBIDDEN | 403 | 无权限 |
| NOT_FOUND | 404 | 资源不存在 |
| INTERNAL_ERROR | 500 | 服务器内部错误 |
